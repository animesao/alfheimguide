# 🔍 GitHub Tracking Troubleshooting

## Возможные причины, почему бот не проверяет GitHub:

### 1. ❌ GITHUB_TOKEN не установлен

**Проверка:**
```bash
# Проверьте файл .env
cat .env | grep GITHUB_TOKEN
```

**Решение:**
```bash
# Добавьте в .env
GITHUB_TOKEN=your_github_token_here
```

**Как получить токен:**
1. Перейдите на https://github.com/settings/tokens
2. Generate new token (classic)
3. Выберите scopes: `repo`, `user`
4. Скопируйте токен

---

### 2. ❌ GitHub модуль отключен

**Проверка:**
```
/status
```
Посмотрите на "Modules Status" - GitHub должен быть ✅

**Решение:**
```
/config module:GitHub action:Enable
```

---

### 3. ❌ Канал уведомлений не установлен

**Проверка:**
```
/status
```
Посмотрите на "Notification Channel" - должен быть канал

**Решение:**
```
/set_channel channel:#notifications
```

---

### 4. ❌ Нет отслеживаемых пользователей

**Проверка:**
```
/status
```
Посмотрите на "Tracked GitHub Users"

**Решение:**
```
/add_user github_username:octocat
```

---

### 5. ❌ Бот не имеет прав на отправку в канал

**Проверка:**
- Проверьте права бота в канале уведомлений
- Бот должен иметь права "Send Messages" и "Embed Links"

**Решение:**
- Дайте боту права в настройках канала

---

### 6. ❌ Фоновая задача не запустилась

**Проверка в логах:**
```
INFO | alfheim_bot | Logged in as BotName
```

Должно быть сообщение о запуске задачи.

**Решение:**
- Перезапустите бота
- Проверьте логи на ошибки

---

## 🔧 Диагностика

### Запустите скрипт диагностики:

```bash
python debug_github.py
```

Этот скрипт покажет:
- ✅ Наличие GITHUB_TOKEN
- 📊 Список отслеживаемых пользователей
- ⚙️ Статус модуля GitHub для каждого сервера
- 📝 Настройки каналов
- 📦 Количество репозиториев

## 📝 Пошаговая настройка

### Шаг 1: Проверьте токен
```bash
# В .env должно быть:
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxx
```

### Шаг 2: Включите модуль
```
/config module:GitHub action:Enable
```

### Шаг 3: Установите канал
```
/set_channel channel:#github-updates
```

### Шаг 4: Добавьте пользователя
```
/add_user github_username:torvalds
```

### Шаг 5: Проверьте статус
```
/status
```

Должно быть:
- ✅ GitHub: Enabled
- ✅ Notification Channel: #github-updates
- ✅ Tracked GitHub Users: torvalds

## 🐛 Частые ошибки

### Ошибка: "User not found"
- Проверьте правильность username
- Убедитесь, что пользователь существует на GitHub

### Ошибка: "Rate limit exceeded"
- GitHub API имеет лимиты
- Подождите или используйте токен с большими лимитами

### Ошибка: "Bad credentials"
- Токен неверный или истёк
- Создайте новый токен

---

## 📊 Как работает проверка

1. **Каждые 2 минуты** бот проверяет GitHub
2. Для каждого отслеживаемого пользователя:
   - Получает список репозиториев
   - Сравнивает с сохранённым состоянием
   - Проверяет новые коммиты
3. Если есть изменения:
   - Отправляет уведомление в канал
   - Обновляет базу данных

---

## 🔍 Проверка в реальном времени

### Проверьте, что задача работает:

1. Добавьте пользователя с активными репозиториями
2. Подождите 2 минуты
3. Проверьте канал уведомлений

### Или создайте тестовый коммит:

1. Отслеживайте свой GitHub
2. Сделайте коммит в любой репозиторий
3. Подождите до 2 минут
4. Проверьте уведомление

## 💡 Советы

### Для тестирования:
- Используйте активных пользователей (torvalds, github, microsoft)

- Отслеживайте только нужных пользователей
- Используйте отдельный канал для уведомлений
- Проверяйте лимиты API
